
## 1. Enums criados na aula

### O que é um `enum` em Java

Um `enum` (enumeração) é um tipo especial de classe que representa um **conjunto fixo e finito de valores**. Ele é usado quando um atributo só pode assumir **opções pré-definidas**, evitando valores inválidos e substituindo o uso de `String` solta no código.

Enums aumentam:
- segurança de tipo
- legibilidade
- padronização
- facilidade de manutenção

### Enum `TipoMotor`

```Java
package curso.arquiteturaspring.montadora;  

public enum TipoMotor {
     ASPIRADO,
    TURBO,     
    ELETRICO 
}
```

`TipoMotor` define **os tipos possíveis de motor** que um carro pode possuir.  
Cada valor é uma constante.
- `ASPIRADO` → motor convencional
- `TURBO` → motor com sobrealimentação
- `ELETRICO` → motor elétrico

### Por que usar enum aqui

Sem enum, seria comum algo como:

```Java
String tipoMotor = "Turbo";
```

Isso é frágil, pois aceita qualquer valor.  
Com enum, o código só aceita valores válidos:

```Java
TipoMotor tipo = TipoMotor.TURBO;
```

## 2. Classe `Carro`

```Java
package curso.arquiteturaspring.montadora;  
import java.awt.*;  

public class Carro {
```

A classe `Carro` representa um **modelo genérico de carro**, servindo como **classe base** para veículos específicos.

Ela define **o que todo carro tem**, mas não define um carro concreto.

### Atributos da classe `Carro`

```Java
private String modelo; 
private Color cor; 
private Motor motor; 
private Montadora montadora;
```

#### Análise dos atributos
- `modelo` → nome do modelo do carro (ex: HRV)
- `cor` → cor do carro (usando `java.awt.Color`)
- `motor` → composição com a classe `Motor`
- `montadora` → enum que indica a fabricante

Aqui aparecem dois conceitos importantes:

- **Composição** → `Carro` _tem um_ `Motor`
- **Enum como regra de negócio** → `Montadora`

### Construtor da classe `Carro`

```Java
public Carro(Motor motor) {     this.motor = motor; }
```

Esse construtor força que **todo carro seja criado com um motor**.
Isso garante que:
- não exista carro sem motor
- o objeto já nasce em um estado válido

Esse padrão é muito comum em arquitetura orientada a objetos e também em Spring (injeção por construtor).

## 3. Classe `HondaHRV`

```Java
package curso.arquiteturaspring.montadora;  
import java.awt.*;  

public class HondaHRV extends Carro {
```

### Conceito principal: Herança

`HondaHRV` **estende** a classe `Carro`.

Isso significa que:
- `HondaHRV` é um `Carro`
- herda todos os atributos e métodos de `Carro`
- especializa o comportamento e as características

Esse é um exemplo clássico de **especialização**.

### Construtor da classe `HondaHRV`

```Java
public HondaHRV(Motor motor) {
     super(motor);     
     setModelo("HRV");     
     setCor(Color.BLACK);     
     setMontadora(Montadora.HONDA); }
```


 ```Java
super(motor);
```

Chama o construtor da classe pai (`Carro`), garantindo que:
- o motor seja corretamente atribuído
- a regra “todo carro tem motor” seja respeitada

```Java
setModelo("HRV");
```

Define o modelo fixo desse carro.  
Esse valor não precisa ser passado externamente, pois faz parte da identidade do `HondaHRV`.

```Java
setCor(Color.BLACK);
```

Define uma configuração padrão de cor para esse modelo.
`
```Java
setMontadora(Montadora.HONDA);
```

Associa explicitamente o carro à montadora `HONDA`, usando o enum criado anteriormente.
