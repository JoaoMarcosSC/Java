
### Configuração do datasource

`spring:   datasource:`

O bloco `datasource` define **como a aplicação vai se conectar ao banco de dados**. A partir dessas informações, o Spring cria automaticamente um **pool de conexões** e disponibiliza o banco para repositórios, DAOs e JPA.

### URL de conexão JDBC
`url: jdbc:postgres://localhost:5432/library`
Essa propriedade define **onde está o banco de dados**.

- `jdbc:postgres`:
    - Indica que o driver utilizado é o **PostgreSQL**
- `localhost`:
    - Endereço do servidor do banco
    - Nesse caso, o PostgreSQL está rodando localmente (via Docker com porta exposta)
- `5432`:
    - Porta padrão do PostgreSQL
- `library`:
    - Nome do banco de dados criado no container

Ou seja, essa URL diz literalmente:

> “Conecte-se ao banco `library`, no PostgreSQL rodando em `localhost`, na porta `5432`.”

### Usuário do banco

`username: postgres`
Define o **usuário de autenticação** no banco de dados.
- Esse usuário foi configurado no momento da criação do container PostgreSQL
- É o usuário administrador padrão do PostgreSQL

### Senha do banco

`password: postgres_password`

Define a **senha do usuário** informado acima.
- Deve ser exatamente a mesma senha passada na variável `POSTGRES_PASSWORD` ao subir o container
- Caso esteja errada, a aplicação falha ao iniciar
### Driver JDBC

`driver-class-name: org.postgresql.Driver`

Aqui é especificado explicitamente **qual driver JDBC será utilizado**.
- `org.postgresql.Driver`:
    - Driver oficial do PostgreSQL
- Permite que o Java saiba **como se comunicar com o PostgreSQL**
- Em muitos casos o Spring Boot detecta automaticamente, mas declarar explicitamente:
    - Evita ambiguidade
    - Torna a configuração mais clara e didática
