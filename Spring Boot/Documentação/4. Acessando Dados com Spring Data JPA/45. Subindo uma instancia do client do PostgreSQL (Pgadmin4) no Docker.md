
## Comando executado

```shell
docker run --name pgadmin4 -p 15432:80 \ -e PGADMIN_DEFAULT_EMAIL=admin@admin.com \ -e PGADMIN_DEFAULT_PASSWORD=admin \ dpage/pgadmin4
```

Esse comando:
- baixa a imagem do pgAdmin
- cria o container
- configura usuário e senha
- expõe a interface web
- sobe o pgAdmin pronto para uso

## `docker run`
- cria um container a partir de uma imagem
- inicia o container imediatamente

Como a imagem `dpage/pgadmin4` não existia localmente, o Docker fez o **pull automático**.

## `--name pgadmin4`

Define o nome do container como `pgadmin4`.

## Mapeamento de portas `-p 15432:80`

Significa:
`porta_do_host : porta_do_container`
- O pgAdmin roda na porta **80** dentro do container
- Essa porta foi exposta como **15432** na máquina local

Assim, o acesso ao pgAdmin é feito em:
`http://localhost:15432`

## Variáveis de ambiente do pgAdmin

Essas variáveis são obrigatórias para inicializar o pgAdmin em **modo servidor**.
### Email do usuário administrador
Define o usuário administrador do pgAdmin (login).

### Senha do usuário administrador
Define a senha para o login.
Essas credenciais são usadas **apenas para acessar o pgAdmin**, não o PostgreSQL.

## Imagem `dpage/pgadmin4`

- Imagem oficial do pgAdmin
- Mantida pela comunidade do PostgreSQL
- Interface web moderna
- Executa via navegador
