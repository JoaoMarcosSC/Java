
## `@Entity`

A anotação `@Entity` indica que a classe Java representa uma **entidade JPA**, ou seja, um objeto que será **mapeado para uma tabela do banco de dados**. Ao marcar uma classe com `@Entity`, o Spring (via JPA/Hibernate) passa a gerenciar o ciclo de vida desse objeto, permitindo operações como persistir, buscar, atualizar e remover registros.

Sem essa anotação, a classe é tratada apenas como um POJO comum e **não participa do contexto de persistência**.
## `@Table`

A anotação `@Table` define explicitamente **qual tabela do banco de dados** a entidade está associada. O atributo `name` indica o nome da tabela. Caso essa anotação não seja usada, o JPA tenta inferir o nome da tabela a partir do nome da classe.

No código apresentado, ela garante que a entidade `Produto` esteja vinculada à tabela `produto`, exatamente como foi criada no `data.sql`.
## `@Id`

A anotação `@Id` identifica o **campo que representa a chave primária** da entidade. Esse campo é usado pelo JPA para identificar de forma única cada registro no banco de dados.

Toda entidade JPA **obrigatoriamente precisa ter um campo anotado com `@Id`**, pois é a partir dele que o Hibernate controla identidade, atualização e exclusão dos registros.

## `@Column`

A anotação `@Column` define o **mapeamento entre um atributo da classe e uma coluna da tabela**. Ela permite configurar detalhes como nome da coluna, nulabilidade, tamanho e unicidade.

Quando o atributo `name` é informado, o mapeamento é explícito. Quando não é informado, o JPA utiliza o nome do atributo como nome da coluna por padrão.

No campo `id`, o nome da coluna é definido explicitamente como `id`, garantindo alinhamento exato com a tabela do banco.`

Nos campos `nome`, `descricao` e `preco`, a anotação `@Column` é utilizada sem parâmetros. Nesse caso, o JPA assume automaticamente que o nome da coluna no banco é igual ao nome do atributo da classe.
## Relação direta com a tabela do banco

Esse conjunto de anotações cria o seguinte mapeamento conceitual:

| Classe Java | Tabela  | Coluna    |
| ----------- | ------- | --------- |
| Produto     | produto | id        |
| Produto     | produto | nome      |
| Produto     | produto | descricao |
| Produto     | produto | preco     |

Esse mapeamento permite que o JPA converta automaticamente:
- **Objeto Java → registro no banco**
- **Registro do banco → objeto Java**

## Conclusão

Essas anotações são a base do **ORM (Object–Relational Mapping)** no Spring:
- `@Entity` transforma a classe em uma entidade persistente
- `@Table` vincula a entidade a uma tabela específica
- `@Id` define a chave primária
- `@Column` mapeia atributos para colunas

Sem esse conjunto mínimo, não é possível utilizar corretamente `Repository`, `save`, `findById` e demais operações de persistência com JPA.
