
## Comando executado

```shell
docker run --name librarydb -p 5432:5432 \ -e POSTGRES_PASSWORD=postgres_password \ -e POSTGRES_USER=postgres \ -e POSTGRES_DB=library \ postgres:16.3
```
## `docker run`

O comando `docker run` é usado para:
- criar um container a partir de uma imagem
- iniciar esse container imediatamente

Se a imagem não existir localmente, o Docker **faz o download automaticamente**.

## `--name librarydb`

Define o **nome do container** como `librarydb`.

Isso permite:
- identificar facilmente o container
- usar comandos como:

Sem esse parâmetro, o Docker geraria um nome aleatório.

## Mapeamento de portas `-p 5432:5432`

Esse parâmetro faz o **bind de portas**:
`porta_do_host : porta_do_container`

Nesse caso:
- PostgreSQL roda na porta `5432` dentro do container
- Essa porta é exposta como `5432` na máquina local

Isso permite que:
- aplicações locais
- IDEs
- ferramentas como DBeaver ou pgAdmin

se conectem ao banco usando:
`localhost:5432`


## Variáveis de ambiente (`-e`)

Essas variáveis são **lidas pela imagem oficial do PostgreSQL** durante a inicialização do container.

### Usuário do banco
`-e POSTGRES_USER=postgres`
Define o **usuário administrador** do banco.

### Senha do usuário
`-e POSTGRES_PASSWORD=postgres_password`
Define a **senha** do usuário criado.

> Essa variável é obrigatória.  
> Sem ela, o container não sobe.

### Banco de dados inicial
`-e POSTGRES_DB=library`
Cria automaticamente um banco chamado `library`.

Ou seja, ao final:
- o usuário `postgres` existe
- a senha está definida
- o banco `library` já está criado

Sem isso, o banco padrão seria `postgres`.

## Imagem `postgres:16.3`

Indica:
- imagem oficial do PostgreSQL
- versão **16.3**

Se a imagem não existir localmente, o Docker:
1. faz o download (`Pulling from library/postgres`)
2. armazena no cache local
3. reutiliza em execuções futuras

## `docker ps`

Resultado:
```shell
CONTAINER ID   IMAGE           STATUS         PORTS b7f474e6a0a8   postgres:16.3   Up About a minute   0.0.0.0:5432->5432/tcp
```

Isso confirma que:
- o container está **em execução**
- a porta 5432 está exposta
- o banco está acessível
